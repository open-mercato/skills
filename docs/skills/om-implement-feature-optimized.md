# om-implement-feature-optimized

> 🧑‍💻 Interactive — acts once, may ask questions, hands control back

Thin alias that runs [om-implement-feature](om-implement-feature.md) with the `optimized` profile: implementation is fanned out to a configured worker model as bounded, file-disjoint packets while the host keeps the spec, integration, and every quality gate.

## Parameters

Same as [om-implement-feature](om-implement-feature.md); the profile is fixed to `optimized`.

## Works with

Wraps [om-implement-feature](om-implement-feature.md). Requires an audited worker binding from [om-setup-agent-harness](om-setup-agent-harness.md).
